# 云海山庄·大气与天气视觉系统

> 文档版本：v1.0 | 状态：设计阶段 | 适用：C++20/SFML 3.0.2 项目
>
> 依赖已有系统：`CloudSystem`（云海状态驱动力）、`GameClock`（昼夜节律时钟）
>
> 设计目标：在现有 `CloudState` 基础上，构建从"云海状态"到"屏幕画面"的完整视觉链路——涵盖昼夜天空渐变、天气粒子特效、大气光晕叠加、雾效深度营造，实现 60FPS 下流畅的大气视觉表现。

---

## 一、设计理念

### 1.1 核心理念："氛围即叙事"

大气与天气不是装饰，而是云海山庄叙事的核心载体：

- 薄雾天气的柔和蓝灰，传达安宁与希望
- 浓云海翻涌的深紫层次，预示着灵气的丰沛
- 大潮时刻的金红霞光，宣告传说事件的降临
- 昼夜交替的天空渐变，让玩家感知时间的流逝与四季的轮回
- 雨/雾/云粒子穿梭于场景之间，赋予世界呼吸感

### 1.2 与现有系统的关系

```
CloudSystem (domain)          GameClock (domain)
        │                            │
        │  current_state_            │  day_ / minutes_in_day_
        │  forecast_state_          │  CurrentPhase()
        │                            │
        └──────────┬─────────────────┘
                   │
                   ▼
        AtmosphereDomain (domain)
        - current_sky_profile_
        - current_weather_profile_
        - transition_progress_
        - daynight_overlay_alpha_
        - fog_density_
        - ambient_light_color_
        - atmosphere_intensity_
                   │
                   ▼
        AtmosphereRenderer (engine)
        - DrawSkyGradient_()
        - DrawWeatherParticles_()
        - DrawAuraOverlay_()
        - DrawFog_()
        - DrawDaynightOverlay_()
                   │
                   ▼
        渲染到 sf::RenderWindow
```

### 1.3 设计目标

| 目标 | 说明 |
|------|------|
| 60 FPS 流畅 | 所有特效在主循环 Update/Render 中完成，粒子对象池复用 |
| 数据驱动 | 天空颜色、粒子参数、光晕强度全部外部配置，无需硬编码 |
| 无缝衔接 | 复用 `ui_layout.json` 的背景色和光晕色，`BgColor_` 和 `AuraColor_` 逻辑不变 |
| 视觉层次 | 天空 → 远景雾 → 场景层 → 天气粒子 → 大气光晕 → UI（6 层） |
| 昼夜循环 | 基于 `DayPhase`（晨/昼/昏/夜）的天空色过渡 |
| 天气过渡 | 云海状态切换时，平滑插值过渡（默认 3 秒） |

## 二、大气视觉层次

```
┌─────────────────────────────────────────────────────┐
│ L5: 大气光晕层 (Aura Overlay)                         │
│     - 场景最顶层全屏半透明叠加                        │
│     - 颜色来自 ui_layout.json 的 aura 字段           │
│     - 透明度随云海状态变化（浓云海/大潮时加深）        │
├─────────────────────────────────────────────────────┤
│ L4: 天气粒子层 (Weather Particles)                   │
│     - 雾气粒子 / 云朵飘动 / 雨滴 / 大潮光尘           │
│     - 使用对象池管理，粒子数 ≤ 200                    │
│     - 按云海状态启用不同粒子类型                       │
├─────────────────────────────────────────────────────┤
│ L3: 场景层 (World Scene)                            │
│     - 已有渲染：地图瓦片、玩家、NPC、灵兽、特效        │
│     - 本系统不修改此层                                │
├─────────────────────────────────────────────────────┤
│ L2: 远景雾效层 (Distance Fog)                       │
│     - 场景边缘的深度雾                               │
│     - 颜色 = 当前天空色的暗色版本                      │
│     - 夜间加深，白天减轻                              │
├─────────────────────────────────────────────────────┤
│ L1: 昼夜天空层 (Day-Night Sky Gradient)             │
│     - 顶部 → 底部线性渐变                             │
│     - 两端颜色 = 当前时间段的天空色（数据驱动）        │
│     - 覆盖全屏，渲染在所有内容下方                     │
├─────────────────────────────────────────────────────┤
│ L0: 背景填充 (Fallback Background)                  │
│     - 即现有 BgColor_(ui_layout_, cloud_state_)      │
│     - 天空渐变层渲染失败时的安全网                     │
└─────────────────────────────────────────────────────┘
```

## 三、数据结构设计

### 3.1 领域层类型 (domain)

```cpp
// 大气状态（运行时数据，与 CloudSystem / GameClock 联动）
namespace CloudSeamanor::domain {

struct AtmosphereState {
    // 当前活跃的天空配置
    SkyProfile current_sky;

    // 当前活跃的天气配置
    WeatherProfile current_weather;

    // 过渡状态
    SkyProfile target_sky;
    WeatherProfile target_weather;
    float transition_duration = 3.0f;  // 过渡持续秒数
    float transition_elapsed = 0.0f;   // 已过渡秒数
    bool is_transitioning = false;

    // 昼夜状态
    float daynight_overlay_alpha = 0.0f;   // 0=白天, 1=深夜
    float fog_density = 0.0f;               // 雾效密度 0~1
    sf::Color ambient_light_color;          // 环境光颜色
    float atmosphere_intensity = 1.0f;     // 大气强度（云海越浓越大）

    // 昼夜天空色插值缓存
    sf::Color sky_top_color;
    sf::Color sky_bottom_color;
    float sun_moon_y_ratio = 0.5f;         // 太阳/月亮在天空的位置
};

}  // namespace CloudSeamanor::domain
```

## 五、技术实现

### 5.1 天空渐变渲染

使用 SFML `sf::VertexArray` + `Quads` 构建全屏双三角渐变：

```cpp
// 顶点顺序（左上→右上→右下→左下）
// 两端颜色 = sky_top_color（顶部）和 sky_bottom_color（底部）
// 使用 clamp((v.texPos.y / windowHeight), 0, 1) 作为插值因子 t
// 最终色 = lerp(sky_bottom, sky_top, t)
```

### 5.2 粒子系统

对象池设计（最多 200 个粒子）：

```cpp
struct AtmosphereParticle {
    sf::Vector2f position;
    sf::Vector2f velocity;
    float size;
    float lifetime;
    float age;
    sf::Color color;
    bool active = false;
};

class ParticlePool {
    static constexpr size_t kMaxParticles = 200;
    std::array<AtmosphereParticle, kMaxParticles> pool_;
    // 复用：取第一个 inactive 的粒子
    AtmosphereParticle* Acquire();
    void Release(AtmosphereParticle* p);
};
```

### 5.3 大气光晕叠加

复用 `ui_layout.json` 的 `aura` 字段，直接使用现有 `AuraColor_` 计算结果：

```cpp
sf::Color aura_color = AuraColor_(ui_layout_config_, cloud_state);
// 透明度根据云海状态调制
float aura_alpha = cloud_state == CloudState::Tide ? 0.35f
                    : cloud_state == CloudState::DenseCloud ? 0.2f
                    : cloud_state == CloudState::Mist ? 0.12f
                    : 0.0f;
```

### 5.4 远景雾效

使用 SFML `sf::Shader`（GLSL）实现屏幕空间雾：

```glsl
// 在 WorldRenderer 中叠加雾效 shader
uniform sampler2D sceneTexture;
uniform vec3 fog_color;
uniform float fog_density;
uniform float time;

void main() {
    vec4 color = texture2D(sceneTexture, gl_TexCoord[0].xy);
    float depth = gl_FragCoord.z / gl_FragCoord.w;  // 线性深度
    float fog_factor = 1.0 - exp(-fog_density * depth);
    fog_factor = clamp(fog_factor, 0.0, 1.0);
    color.rgb = mix(color.rgb, fog_color, fog_factor);
    gl_FragColor = color;
}
```
