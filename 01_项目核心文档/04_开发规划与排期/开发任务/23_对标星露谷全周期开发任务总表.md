## 二、开发总策略

### 2.1 核心原则

1. **先打通循环，再填充内容**：每个阶段必须有可验证的核心闭环
2. **数据驱动优先**：配置表外置，代码只写通用逻辑
3. **保持差异化**：云海/灵茶/灵兽/契约是超越SDV的核心
4. **无压力设计**：错过可补、不会破产、战斗可选
5. **小步快跑**：每周有可运行版本，每月有可演示版本

### 2.2 阶段划分总览

```
Phase 0: 技术清债期 (M1)        → 工程化底座
Phase 1: 核心闭环期 (M2-M3)     → 可连续游玩7天
Phase 2: 系统扩展期 (M4-M6)     → 多路径成长
Phase 3: 内容爆发期 (M7-M9)     → 对标SDV内容量
Phase 4: 打磨发布期 (M10-M12)   → EA发布准备
```

### 2.3 年度里程碑

| 里程碑 | 时间 | 交付标准 |
|--------|------|---------|
| M1 | 月1末 | 架构重构完成，测试覆盖率≥60%，GameApp<250行 |
| M2 | 月2末 | 核心闭环可玩：种植→加工→售卖→送礼→好感 |
| M3 | 月3末 | Internal Demo v0.2：3角色+8作物+对话+UI |
| M6 | 月6末 | External Demo v0.5：6角色+灵界+灵兽+节日 |
| M9 | 月9末 | Beta v0.8：12角色+战斗+钓鱼+烹饪+主线 |
| M12 | 月12末 | EA v1.0：完整内容+优化+Steam发布 |

## 九、关键建议与风险提示

### 9.1 核心建议

| 建议 | 理由 | 优先级 |
|------|------|--------|
| **美术优先外包/素材包** | 自绘像素风需要6-12个月，素材包1-2周 | 🔴 关键 |
| **对话内容AI辅助+人工润色** | 12角色×50条对话=600+条，纯手写耗时巨大 | 🔴 关键 |
| **先打通循环再填充内容** | 空有内容无循环=不好玩，有循环缺内容=可迭代 | 🔴 关键 |
| **保持云海/灵茶/灵兽差异化** | 这是超越SDV的核心，不能沦为SDV翻版 | 🟡 重要 |
| **尽早玩家测试** | Phase 1完成后立即找目标用户测试 | 🟡 重要 |
| **音频使用免版税/AI生成** | 原创配乐成本高，MVP阶段可用替代方案 | 🟡 重要 |
| **数据驱动一切** | 配置外置=平衡调整改表即可=迭代速度快 | 🟡 重要 |
| **每周有可运行版本** | 防止长时间开发后集成灾难 | 🟢 建议 |

### 9.2 风险矩阵

| 风险 | 概率 | 影响 | 缓解策略 |
|------|------|------|----------|
| 美术资源制作周期过长 | 高 | 致命 | 外包/素材包/分批交付 |
| 内容量不足（对话/事件） | 高 | 严重 | AI辅助生成+人工润色 |
| 重构引入回归Bug | 中 | 严重 | 先写测试再重构+小步提交 |
| 工期估算不足 | 中 | 中等 | 每Task预留20%缓冲 |
| 经济平衡失调 | 中 | 中等 | 数据驱动+自动化模拟测试 |
| 个人精力不足 | 中 | 严重 | 严格优先级+砍P2内容 |
| 过度设计 | 低 | 中等 | YAGNI原则+只解决当前痛点 |

### 9.3 预算估算

| 项目 | 方案A（低成本） | 方案B（中等投入） |
|------|----------------|-----------------|
| 美术素材 | $50-100（素材包） | $500-2000（外包定制） |
| 音频素材 | $0-50（免版税） | $200-500（定制BGM） |
| 对话内容 | $0（AI+自写） | $100-300（外包润色） |
| Steam发布费 | $100 | $100 |
| **总计** | **$150-250** | **$900-2900** |

## 十、每阶段统一验收五问

每增加一个系统，都必须回答以下5个问题。如果有2个以上答不上来，该功能不应立即进入开发：

1. **它和哪个现有系统形成闭环？**
2. **玩家是否能在1-2天内感知到收益？**
3. **它是否已经接入存档、日切、UI提示？**
4. **它是否可以配置化/数据驱动？**
5. **它是否强化了"云海山庄"的差异化，而不是只是复制SDV？**
