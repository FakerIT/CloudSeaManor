## 🎨 任务T3.2：获取/制作玩家精灵图

### 需求规格

**动画需求**：
- 8个方向：上、下、左、右、左上、右上、左下、右下
- 每方向4帧行走循环
- 1帧静止站立
- 总计：(8方向 × 4帧行走) + (8方向 × 1帧静止) = **40帧**

**精灵图布局**：
```
 spritesheet.png (480 x 384 px = 10列 x 8行, 每48x48)

 行0-3:  下、上、左、右 的行走动画（每行4帧）
 行4-7:  左下、右下、左上、右上 的行走动画（每行4帧）

 或者更常见的布局：
 行0:   静止 下 上 左 右 左下 右下 左上 右上 (9帧)
 行1-4: 行走动画 下(4帧) 上(4帧) 左(4帧) 右(4帧)
 行5-8: 行走动画 左下(4帧) 右下(4帧) 左上(4帧) 右上(4帧)
```

**配色方案建议**：
- 主色调：暖色系（橙/棕/米白）- 符合东方田园风格
- 头发：深棕色或黑色
- 服装：宽松的棉麻材质感（浅绿/米色/浅棕）
- 配饰：简单茶农装饰（草帽可选）

**Prompt示例**：

**后处理**：
- AI生成的需要手动拆分帧

- 确保透明背景正确

#### 方案C：自制（最灵活但耗时）

**工具推荐**：
- **Aseprite** ($19.99) - 专业像素画软件，**强烈推荐**
  - 动画时间线
  - 图层系统
  - 自动导出精灵表
  - 调色板管理
  
- **LibreSprite** (免费) - Aseprite的开源替代
- **Piskel** (在线免费) - 浏览器中使用，适合初学者
- **GraphicsGale** (免费) - Windows下的选择

**制作流程**：
1. 绘制静止帧（1个方向）
2. 复制并修改为其他7个方向
3. 为每个方向添加3帧行走动画（腿/手臂摆动）
4. 导出为精灵表PNG
5. 在游戏中测试调整

**预计时间**：有经验者15-20小时，新手40-60小时

### 技术实现

```cpp
// === SpriteAnimator 组件 ===
struct AnimationFrame {
    sf::IntRect texture_rect;  // 在精灵表中的位置
    float duration;            // 该帧持续时间（秒）
};

struct Animation {
    std::string name;
    std::vector<AnimationFrame> frames;
    bool loop = true;
    float speed_multiplier = 1.0f;
};

class SpriteAnimator {
public:
    void LoadSpritesheet(const sf::Texture& texture, int frame_width, int frame_height);

    void AddAnimation(const std::string& name, const Animation& anim);
    void Play(const std::string& name);
    void Update(float delta_time);

    void SetFacing(FacingDirection direction);

    const sf::IntRect& GetCurrentFrameRect() const;

private:
    const sf::Texture* spritesheet_ = nullptr;
    int frame_width_ = 48;
    int frame_height_ = 48;

    std::unordered_map<std::string, Animation> animations_;
    Animation* current_anim_ = nullptr;
    int current_frame_index_ = 0;
    float frame_timer_ = 0.0f;

    FacingDirection facing_ = FacingDirection::Down;
};
```

- 1方向静止站立：1-2帧（idle呼吸动画可选）
- 特殊动作（可选）：
  - 工作/劳动动作（阿茶采茶、老槐打坐...）
  - 表情变化（开心、生气、害羞...）
  - 特殊事件动作（送礼、拥抱...）

**总计估算**：
- 基础动画：(4方向 × 4帧) + 2帧静止 = 18帧/NPC
- 12个NPC × 18帧 = **216帧**
- 加上特殊动作可能达到 **300-400帧**

### 批量生产策略

#### 策略1：模板法（高效）

**步骤**：
1. 创建1-2个基础身体模板（男/女）
2. 为每个NPC定制：
   - 发型（5-10分钟）
   - 服装颜色（2分钟）
   - 配饰（5分钟）
3. 批量应用动画帧

**优点**：速度快，风格统一（3-4天完成全部12个）  
**缺点**：多样性有限

#### 策略2：差异化法（质量优先）

**步骤**：
1. 为主要角色（阿茶、小满、晚星、秋棠）精心设计
2. 次要角色使用简化版
3. 背景角色（老槐、锦时）可以更简单

**时间分配**：
- 主要角色（4个）：每人8-10小时 = 32-40小时
- 次要角色（5个）：每人4-6小时 = 20-30小时
- 背景角色（3个）：每人2-3小时 = 6-9小时
- **总计：58-79小时**

#### 策略3：混合法（推荐）

**分配**：
- 4个核心角色自制（确保质量和独特性）
- 5个次要角色修改免费模板
- 3个背景角色使用简化模板

**预计总时间**：30-50小时

   - 只渲染视口内的瓦片（视锥剔除）
   - 批量渲染相同图集的瓦片
   - 缓存静态图层（如果不变化）

4. **额外功能**：
   - 从TMX读取对象点（出生点、传送点、触发区域）
   - 支持瓦片翻转/旋转（TMX内置功能）
   - 支持地形自动拼接（Auto-tile/Auto-mapping）

## ⚠️ 常见陷阱

### 陷阱1：资源过度膨胀

**症状**：assets文件夹 > 1GB，加载时间 > 30秒

**解决**：
- 使用压缩格式（OGG代替WAV，JPG用于大背景）
- 实现资源流式加载（按需加载，而非启动时全加载）
- 定期清理未使用的资源
- 使用Git LFS管理大文件

### 陷阱2：风格不统一

**症状**：玩家是Q版，NPC是写实，UI是扁平化

**解决**：
- 制定明确的**美术风格指南**（Art Style Guide）
- 所有资源参考同一套调色板
- 固定像素密度（如统一32px基准）
- 定期全局审查视觉一致性

### 陷阱3：忘记适配不同分辨率

**症状**：在高分屏上UI太小，低分屏上UI太大

**解决**：
- 使用**虚拟坐标系统**（如1920x1080内部分辨率）
- 自动缩放到实际窗口大小
- UI使用相对坐标（百分比）而非绝对像素
- 测试多种分辨率（720p, 1080p, 1440p）
