#### 问题5：错误处理不够健壮 ⚠️⚠️⚠️

**问题描述**：

**问题代码示例**：

**改进方案**：

##### 使用std::expected（C++23）或自定义Result类型

```cpp
// 定义统一的错误枚举
enum class InventoryError {
    InvalidItemId,
    InvalidCount,
    InventoryFull,
    ItemNotFound,
    InsufficientQuantity
};

// 使用std::expected（如果编译器支持C++23）
#include <expected>

std::expected<void, InventoryError> Inventory::AddItem(const std::string& item_id, int count) {
    if (item_id.empty()) return std::unexpected(InventoryError::InvalidItemId);
    if (count <= 0) return std::unexpected(InventoryError::InvalidCount);
    if (slots_.size() >= kMaxSlots) return std::unexpected(InventoryError::InventoryFull);

    // 正常逻辑...
    return {};
}

// 调用方式
auto result = inventory.AddItem("Wood", 5);

    Logger::Error("AddItem failed: {}", static_cast<int>(result.error()));
    push_hint("背包已满！", 2.0f);
}
```

##### 或使用简单的bool+错误码模式（兼容C++20）

```cpp
struct Result {
    bool success;
    std::string error_message;
};

Result Inventory::AddItemSafe(const std::string& item_id, int count) {
    if (item_id.empty()) {
        return {false, "物品ID不能为空"};
    }
    if (count <= 0) {
        return {false, "数量必须为正数"};
    }

    AddItem(item_id, count);  // 内部实现不变
    return {true, ""};
}
```

### 7.3 星露谷类游戏的特殊注意事项

#### 设计层面
1. **节奏控制**：确保新手教程平缓（你做得不错！已有引导系统）
2. **反馈及时**：每个操作都要有视觉/音效反馈（目前只有文字提示）
3. **目标明确**：短期/中期/长期目标清晰（契约系统很好）
4. **容错性强**：允许玩家犯错但不严厉惩罚（保护期机制很好）

#### 技术层面
1. **存档安全**：防止玩家篡改存档获取不当利益
2. **模组友好**：预留扩展点（事件系统、数据驱动）
3. **性能预算**：确保低端设备也能稳定运行
4. **调试功能**：保留开发者控制台（F3/F5/F7很好）
