# 【系统】灵茶图鉴与茶灵收集（V1.0）

**日期**：2026-04-28  
**状态**：设计草稿 / 可进入原型拆分  
**定位**：围绕“制茶成就 -> 茶灵现身 -> 图鉴收藏 -> 庄园陈列”建立长期收集线，放大灵茶主题的辨识度与持续更新价值

---

## 1. 设计目标

### 1.1 核心价值

灵茶系统已经是《云海山庄》的核心经济与审美主题，但当前玩家对“做出高品质茶”的长期追求仍偏功能性。茶灵系统的目标是把这条线从收益驱动升级为收藏驱动与情感驱动。

### 1.2 系统目标

- 为每种代表性灵茶提供一个可收集、可展示、可讨论的“茶灵”
- 将高品质制茶与特殊时机饮茶转化为具有惊喜感的收藏事件
- 让图鉴成为茶室内的长期目标展示面板
- 为后续节日、邮件、庄园装饰、隐藏事件提供持续扩展入口

### 1.3 设计原则

- **依附核心玩法**：不新增独立收集操作，收集自然发生在制茶和饮茶流程中
- **先显性、后隐藏**：基础茶灵容易理解，稀有茶灵保留探索空间
- **收藏可见**：解锁后必须在庄园中有实体感，不只是一页菜单
- **治愈优先**：收集失败不惩罚，只意味着“这次缘分未到”

---

## 2. 核心体验循环

```text
种植与制茶
  -> 产出高品质灵茶
  -> 在对应季节 / 时辰 / 场景饮用或陈放
  -> 茶灵现身并加入图鉴
  -> 获得图鉴奖励 / 茶室陈列 / NPC 评论
  -> 激励玩家继续完善茶谱与品质追求
```

### 2.1 首个惊喜时刻

1. 玩家制出第一份 `圣` 级灵茶
2. 在茶室或特定场景饮用时，触发短暂现身演出
3. 茶灵围绕茶盏盘旋，留下图鉴条目与一句短描述
4. 茶室“灵茶架”新增该茶灵的可视化摆件

## 3. 茶灵体系设计

### 3.1 茶灵定义

茶灵是灵茶在高品质、合时令、合心境条件下凝出的微型灵体。它们不承担战斗宠物职能，首版定位是：

- 图鉴收藏物
- 茶室装饰表现
- 触发少量评论与隐藏线索的叙事载体

### 3.2 茶灵分层

| 层级 | 数量建议 | 解锁特征 | 设计用途 |
|---|---|---|---|
| 常规茶灵 | 8-12 个 | 圣品茶 + 合适时辰 | 作为基础收集线 |
| 季候茶灵 | 4-6 个 | 额外季节/天气限制 | 增强探索感 |
| 隐藏茶灵 | 2-4 个 | 稀有组合条件 | 提供社区话题 |
| 祥瑞茶灵 | 1 个 | 图鉴近满或生态大成 | 终局展示奖励 |

### 3.3 示例映射

| 灵茶 | 茶灵形象 | 基础解锁条件 |
|---|---|---|
| 初雾绿茶 | 翠色小龙 | 春季清晨饮用圣品初雾绿茶 |
| 晨露白茶 | 晶露小鹿 | 雨后清晨饮用圣品晨露白茶 |
| 暮霞乌龙 | 霞翼燕 | 夏季黄昏饮用圣品暮霞乌龙 |
| 秋霜乌龙 | 霜尾狐 | 秋季夜晚饮用圣品秋霜乌龙 |
| 冬雪银针 | 雪羽鹤 | 冬季雪天饮用圣品冬雪银针 |
| 云巅金芽 | 金芽灵雀 | 高灵气日于茶室静坐后饮用 |

## 4. 解锁机制

### 4.1 基础条件

基础版采用“双钥匙”规则：

1. 玩家成功获得指定灵茶的 `圣` 级品质成品
2. 在正确的季节/时辰/场景完成饮用或供奉行为

### 4.2 条件结构建议

```cpp
struct TeaSpiritUnlockRule {
    std::string tea_id;
    std::string tea_spirit_id;
    std::string required_quality;     // Holy
    std::string season;
    std::string time_range;
    std::string weather_limit;
    std::string location_id;
    std::string extra_condition;      // eco_balanced / first_brew / festival_day
};
```

### 4.3 失败体验设计

- 条件不满足时只表现为“平静饮茶”，不提示失败
- 若接近触发条件，可由 NPC、日记或环境描述给出朦胧暗示
- 已触发过的茶灵不再消耗高稀有成本条件，可允许重复召见用于观赏

## 5. 图鉴与展示系统

### 5.1 图鉴定位

图鉴不只是列表页，而是茶室中的“灵茶架”与“藏灵册”双入口：

- `藏灵册`：系统性查看收集进度、描述、解锁条件提示
- `灵茶架`：将已获得茶灵以实体摆件方式陈列在茶室中

### 5.2 图鉴条目内容

每个条目建议包含：

- 茶灵名称
- 对应灵茶
- 稀有度
- 外观剪影 / 解锁后彩色像素图
- 一句诗性描述
- 首次现身日期
- 再访方式提示（解锁后可见）

### 5.3 图鉴奖励

| 进度里程碑 | 奖励 |
|---|---|
| 收集 3 个 | 茶室小型展示底座 |
| 收集 6 个 | 茶饮体力恢复小幅提升 |
| 收集 12 个 | 茶灵评论事件池解锁 |
| 全图鉴 | `茶灵大师` 称号 + 庄园永久灵气加成 |

## 6. 庄园与其他系统联动

### 6.1 茶室陈列

- 已收集茶灵可显示在茶室指定展示位
- 部分茶灵在夜晚会有极轻微待机动画
- 茶室舒适度或美观度系统可为其提供小额加分

### 6.2 动态叙事联动

- 首次解锁茶灵时，触发特定 NPC 评论或信件
- 图鉴里程碑可以写入“行为记忆系统”

### 6.3 庄园生态联动

- 个别季候茶灵要求庄园元素接近平衡
- 生态极佳时，稀有茶灵的提示概率提升

### 6.4 DIY 联动

- 茶灵本体不可自由摆放到所有区域，避免破坏珍贵感
- 可解锁“茶灵风格摆件”作为装饰变体，与正式收集体分离

## 10. 演出与提示设计

### 10.1 现身演出要求

- 时长控制在 3-5 秒
- 不强制打断玩家太久
- 重点突出光效、茶香粒子、微型灵体剪影

### 10.2 提示策略

- 未解锁时只给“隐约暗示”
- 已解锁后可在图鉴中看到较明确的再现提示
- 稀有隐藏茶灵仍只给半提示，保留讨论空间

## 12. 风险与约束

### 12.1 风险

- 若解锁条件过于玄学，会导致玩家挫败或只能查攻略
- 若茶灵数量扩张过快，会产生高维护成本
- 若展示层表现太薄弱，图鉴价值会被削弱

### 12.2 控制策略

- 首版只上线少量、强识别度茶灵
- 每个茶灵至少绑定一条世界内提示来源
- 先确保“收集瞬间”和“茶室陈列”足够有记忆点，再扩数量

---

## 13. 验收标准

- [ ] 玩家首次达成条件时，茶灵可稳定现身并写入图鉴
- [ ] 图鉴进度、首次解锁日期与奖励领取状态可正确存档
- [ ] 茶室能展示已收集茶灵，读档后状态不丢失
- [ ] 未满足条件时不会出现误提示或明显失败提示
- [ ] 收集 3 个与收集 6 个的里程碑奖励可正常触发
