# 【系统】庄园灵气生态（V1.0）

**日期**：2026-04-28  
**状态**：设计草稿 / 建议先做轻量 MVP  
**定位**：把庄园从静态产地升级为会呼吸、会偏向、会回应玩家经营方式的灵气生态体，让“种什么、净化什么、摆什么”都留下长期痕迹

---

## 1. 设计目标

### 1.1 核心问题

现有系统已经具备灵气、元素、净化、种植与装饰等基础概念，但玩家对“庄园整体气息是否被自己改变”这一层感知还不够强。经营行为之间的联系更多体现为单点收益，而非生态层反馈。

### 1.2 系统目标

- 让玩家长期经营行为在庄园层形成可被观察的生态倾向
- 让元素体系从战斗/设定概念延伸为可感知的经营变量
- 用“失衡产生新机会、平衡带来祥瑞”的方式维持正向循环
- 为灵茶品质、浊灵频率、茶灵收集、NPC 评论提供统一环境基底

### 1.3 设计原则

- **非惩罚型**：失衡不是负债，而是另一种生态状态
- **轻可视化**：不用复杂仪表盘压迫玩家，以环境描述与少量 UI 提示为主
- **可调可救**：玩家永远能通过经营、净化、装饰把生态拉回想要的方向
- **长期缓变**：生态变化以“天”为单位，不追求瞬时剧烈波动

---

## 2. 核心体验循环

```text
玩家持续种植 / 净化 / 装饰 / 饮茶 / 经营
  -> 庄园元素比例与灵气浓度变化
  -> 生态状态改变作物品质、事件权重、访客反馈
  -> 玩家观察庄园“气息”与收益变化
  -> 调整布局与节奏
  -> 追求偏向生态或平衡生态
```

### 2.1 目标体验

玩家应逐渐形成这样的感受：

- “我最近种太多偏风的茶，庄园的气息真的轻了起来。”
- “这几天净化了不少潮系浊灵，后院终于稳下来了。”
- “我想把庄园慢慢调回五行平衡，试试看能不能引来祥瑞。”

---

## 3. 生态指标设计

### 3.1 三个核心指标

| 指标 | 含义 | 玩家体感 |
|---|---|---|
| 灵气浓度 | 庄园整体生命力与灵性强度 | 作物长势、光效、气氛更明显 |
| 元素平衡 | 云 / 风 / 露 / 霞 / 潮 五类元素的相对占比 | 庄园偏向、事件类型、浊灵分布变化 |
| 灵兽活跃度 | 灵兽与庄园互动意愿 | 小帮助、小惊喜、稀有发现概率 |

### 3.2 显示策略

- 默认不展示精确数值，只给状态描述与简图
- 茶室或日记页可查看较完整生态摘要
- 高阶道具或师承技能可解锁更清晰观察能力

### 3.3 生态状态分段

#### 灵气浓度

- 枯静：偏低，庄园显得沉寂
- 平稳：常态区间
- 充盈：适合高品质茶事
- 澄明：高阶状态，可触发祥瑞类机会

#### 元素平衡

- 单元素偏盛
- 双元素共振
- 近似平衡
- 完美平衡

## 4. 生态变化来源

### 4.1 种植影响

| 行为 | 生态影响 |
|---|---|
| 大量种植同元素灵茶 | 该元素占比逐日抬升 |
| 连续收获高品质灵茶 | 灵气浓度小幅提升 |
| 长期忽视庄园整理 | 灵气浓度缓慢下滑 |

### 4.2 净化影响

| 行为 | 生态影响 |
|---|---|
| 净化某元素浊灵 | 对应失衡压力下降 |
| 连续净化成功 | 灵气浓度短期回升 |
| 长时间不处理浊灵 | 某元素异常扩散 |

### 4.3 装饰影响

| 装饰类型 | 影响方向 |
|---|---|
| 风铃、轻幡 | 提升风系权重 |
| 水钵、流泉 | 提升露/潮系权重 |
| 晨光镜、霞灯 | 提升霞系权重 |
| 石景、古木 | 提供平衡稳定效果 |

### 4.4 灵兽与特殊行为

- 灵兽巡园可小幅稳定波动
- 节日供茶、心灵静坐、特殊茶会可给予临时生态增益

---

## 5. 生态反馈设计

### 5.1 对作物与灵茶的影响

- 灵气浓度高：提高生长稳定性与高品质概率
- 单元素偏盛：对对应元素灵茶有加成，对相克方向的茶收益转弱
- 平衡生态：不追求极端收益，但更容易触发祥瑞与稀有事件

### 5.2 对浊灵与净化的影响

- 失衡时，对应元素浊灵出现频率上升
- 这不是纯惩罚，因为也意味着：
  - 更多净化收益机会
  - 更多回流灵气来源
  - 某些材料更容易获得

### 5.3 对 NPC 与日常氛围的影响

- NPC 会评论庄园“风声轻了”“雾气湿了”“霞光更暖了”
- 高生态状态提升某些访客事件权重
- 特定角色更偏好某类庄园气质

### 5.4 对茶灵与收藏的影响

- 某些季候茶灵要求庄园达到近似平衡
- 某些隐藏茶灵需要明显偏向某元素

---

## 6. 祥瑞事件设计

### 6.1 触发条件

祥瑞不是每日目标，而是高质量经营后的偶发回应。建议触发条件采用组合判定：

- 生态达到“澄明 + 近似平衡”持续 2-3 天
- 或某单元素偏向达到高阈值并稳定

### 6.2 事件池示例

| 事件 | 条件倾向 | 结果 |
|---|---|---|
| 茶灵自来 | 平衡生态 | 某稀有茶灵主动现身 |
| 一夜丰熟 | 灵气澄明 | 部分作物成熟推进 |
| 稀客来访 | 平衡或高霞 | 特殊 NPC/旅人出现 |
| 灵兽献礼 | 高活跃度 | 灵兽带来种子或材料 |

### 6.3 设计约束

- 祥瑞是奖励，不是玩家必须刷出的资源点
- 触发频率应偏低，避免变成固定收益路线

## 7. 数据结构建议

### 7.1 domain 模型

```cpp
enum class ManorElement {
    Cloud,
    Wind,
    Dew,
    Glow,
    Tide
};

struct ManorEcologyState {
    int aura_level = 0;
    int spirit_beast_activity = 0;
    std::array<int, 5> element_values {};
    int last_balance_score = 0;
    int balanced_days_streak = 0;
};

struct EcologyDeltaEvent {
    std::string source_type;      // planting / purification / decoration / festival
    std::string source_id;
    ManorElement primary_element;
    int aura_delta = 0;
    int element_delta = 0;
};
```

### 7.2 派生计算

- `balance_score`：五元素差值离散度
- `dominant_element`：当前主导元素
- `ecology_stage`：枯静 / 平稳 / 充盈 / 澄明

## 9. 工程落点建议

### 9.1 domain 层

- 新增 `ManorEcologySystem`
- 负责：
  - 生态数值更新
  - 平衡与偏向计算
  - 祥瑞触发判定

### 9.2 engine 层

- 在日切主入口统一结算生态变化
- 向 HUD、日记、茶室面板提供轻量状态文本
- 驱动环境表现：粒子、色调、音景细节

### 9.3 infrastructure 层

- 负责效果表、事件表、评论表的加载
- 负责生态存档读写与版本兼容

### 9.4 与现有系统的耦合

- 灵茶系统：种植、品质、图鉴
- 战斗净化系统：净化结果反馈
- DIY/装饰系统：摆件对生态微调
- 动态叙事系统：NPC 对生态变化发表评论

## 11. UI 与表现建议

### 11.1 轻 UI 方案

- 茶室面板显示一句生态摘要：
  - “庄园灵气平稳，风意略盛。”
  - “五气相和，似有祥瑞将至。”
- 日记系统可追加生态观察条目

### 11.2 环境表现

- 风系偏盛：飘带、风铃、叶片动态更明显
- 潮系偏盛：空气湿润、薄雾与水声更重
- 霞系偏盛：黄昏色调更暖
- 平衡生态：整体色调柔和、音景更宁静

## 12. MVP 范围与分阶段实施

### 12.1 P0（建议先做）

- 只实现 `灵气浓度 + 五元素权重`
- 只接入 3 类输入：
  - 灵茶种植/收获
  - 浊灵净化
  - 装饰摆件
- 只提供 3 类输出：
  - 灵茶品质权重修正
  - NPC 评论钩子
  - 低频祥瑞事件

### 12.2 P1

- 接入灵兽活跃度
- 增加环境表现与更多祥瑞事件
- 与茶灵收集条件正式打通

### 12.3 P2

- 允许特殊建筑或师承流派影响生态观测能力
- 增加更完整的生态历史回顾页

---

## 13. 风险与约束

### 13.1 风险

- 数值太重会把治愈经营推向“算表”
- UI 太多会制造管理压力
- 反馈太弱则玩家感知不到系统存在

### 13.2 控制策略

- 首版避免显示具体百分比与公式
- 先让环境描述和 2-3 个明确后果跑通
- 平衡态与偏向态都要有价值，防止出现单一最优解

---

## 14. 验收标准

- [ ] 连续种植某元素灵茶后，生态摘要会出现对应偏向描述
- [ ] 净化对应元素浊灵后，偏向可被逐步修正
- [ ] 生态状态能影响至少 1 项灵茶品质结果与 1 项 NPC 评论
- [ ] 读档后生态状态、平衡天数与事件冷却不丢失
- [ ] 完美平衡或高偏向生态可触发至少 1 个祥瑞事件
