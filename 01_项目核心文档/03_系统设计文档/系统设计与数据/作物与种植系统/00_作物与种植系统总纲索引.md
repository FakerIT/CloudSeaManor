> **前期靠农作物站稳脚跟，中期靠灵茶建立经济护城河，后期靠加工链和社交礼赠放大价值。**

## 跨系统联动总览

```
CloudSystem (云海天气)
    │
    ├──► CropGrowthSystem  (生长加速 / 品质提升)
    │         │
    │         └──► TeaPlot.quality  (晴→普通 / 薄雾→优质 / 浓云→稀有 / 大潮→灵品+)
    │
    ├──► WorkshopSystem   (加工速度 / 成功率)
    │
    ├──► SpiritBeastSystem (灵兽状态 / 稀有掉落)
    │
    └──► CloudSeaTideEvent (大潮触发终局剧情)

FarmingSystem (地块操作)
    │
    ├──► Inventory         (消耗种子 / 产出作物)
    │
    ├──► StaminaSystem     (操作消耗体力)
    │
    ├──► SkillSystem       (农业技能经验 / 升级)
    │
    └──► SocialSystem      (送礼 / 好感度)

Inventory (物品系统)
    │
    ├──► 饱食度系统        (食材消耗 → 体力恢复)
    │
    ├──► WorkshopSystem   (加工原料 → 产物)
    │
    └──► PriceTable       (卖出价格 / 品质倍率)

WorkshopSystem (工坊加工)
    │
    ├──► 初加工链          (鲜叶 → 茶叶包)
    │
    └──► 精加工链          (茶叶包 → Buff物品/礼品)

SocialSystem (社交系统)
    │
    ├──► Gift_Preference.json  (角色喜好灵茶)
    │
    └──► Heart Event Trigger   (送礼 → 好感解锁剧情)
```

## 共享设计规则

### 生长时间换算
- 游戏内 **1 现实秒 = 1 游戏分钟**
- **1 游戏天 = 24 小时（1440 分钟）**
- 作物生长时间以**游戏天**为单位，CSV 中的 `growth_time` 字段单位为**秒**
- `growth_time(秒) ÷ 60 = 实际游戏天数`

### 季节规则
- **每季 28 天**，四季共 112 天 = 1 游戏年
- 不适季作物：**自动枯萎**（`ShouldPlotWiltBecauseSeason`）
- 温室地块：**免疫季节枯萎**
- 季节标签：`spring` / `summer` / `autumn` / `winter` / `all`

### 体力消耗
- 整地/播种/浇水/收获各有独立体力消耗（`stamina_cost`）
- 新手保护期（前14天）：体力消耗 × 0.7
- 所有消耗通过 `StaminaSystem.Consume()` 处理

## 品质倍率总表

| 品质 | 产出倍率 | 触发条件 | 售价倍率（相对普通） |
|------|---------|---------|-----------------|
| 普通（Normal） | 1.0× | 晴朗 | 1.0× |
| 优质（Fine） | 1.5× | 薄雾稳定 | 1.5× |
| 稀有（Rare） | 2.0× | 浓云海稳定 | 2.0× |
| 灵品（Spirit） | 3.0× | 大潮 + 高灵气 | 3.0× |
| 圣品（Holy） | 5.0× | 大潮 + 高灵气 + 茶魂花地块 | 5.0× |

### 品质判定逻辑（`CropData.hpp`）

```
cloud_density = CloudSystem.CurrentSpiritDensity()
                Clear=0.0 / Mist=0.3 / DenseCloud=0.7 / Tide=1.0

if cloud_density >= 1.0:       // 大潮
    if aura >= 600:             quality = Spirit(灵品)  [圣品需额外条件]
    else:                       quality = Rare(稀有)
elif cloud_density >= 0.7:      // 浓云海
    quality = Rare(稀有)
elif cloud_density >= 0.3:      // 薄雾
    quality = Fine(优质) 或 Normal(普通)  [各50%]
else:                           // 晴朗
    quality = Normal(普通)
```
