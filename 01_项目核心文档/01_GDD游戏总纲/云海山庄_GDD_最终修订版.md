# 《云海山庄》完整游戏设计文档（GDD）最终修订版

**版本**：2.0（最终修订整合）  
**日期**：2026年4月  
**游戏类型**：治愈系生活模拟 + 轻度奇幻冒险 + 农场经营  
**平台**：PC（Steam优先）/ Nintendo Switch / 后续移动端  
**核心一句话**：在云雾缭绕的百年山庄里，重新定义“家”，守护灵气与人心。  

---

## 1. 项目总览

### 1.1 目标用户
- 25-40岁，偏好《星露谷物语》《动物森友会》《牧场物语》。
- 追求放松、情感共鸣、收集与长期成长。

### 1.2 设计哲学（对标SDV）
- **简单循环**：每天知道做什么。
- **无压力成长**：错过可补，不会破产。
- **系统互连**：种田、社交、探索、经营相互增益。
- **长期目标**：契约系统驱动中后期持续投入。

### 1.3 核心体验
- 从荒废山庄到理想家园。
- 从个人修复到群体守护。
- 从日常经营到终局抉择。

---

## 2. 核心每日循环（最终版）

**1游戏日 = 20分钟真实时间（0.5x~3x可调）**

| 时间段 | 推荐活动 | 玩家收益 |
|---|---|---|
| 清晨 6:00 | 查看云海预报 + 安排日程 | 明确当天重点 |
| 上午 6:00-10:00 | 农活（种植/采摘/喂灵兽） | 作物、灵茶、材料即时入账 |
| 中午 10:00-14:00 | 经营与探索（修缮/工坊/灵界） | 资源与建筑推进 |
| 下午 14:00-18:00 | 社交与采集 | 好感、剧情、任务解锁 |
| 傍晚 18:00-22:00 | 节日/温泉/互动 | 好感、活动奖励 |
| 夜晚 22:00-24:00 | 收尾制作与放松 | 灵气提升、次日增益 |

### 每日UI引导
- 右上角固定“今日推荐三件事”（生产/社交/养成各1条）。
- 右下角“今日小贴士”提示次日云海与建议行动。

### 新手保护
- 前7天手把手任务链。
- 前14天体力消耗-30%，作物不枯萎。

## 3. 云海系统（GDD 3.1 最终优化）

### 3.1 状态与收益（全正向）
| 状态 | 基础概率 | 主要效果 |
|---|---:|---|
| 晴朗 | 40% | 基础正常收益 |
| 薄雾 | 35% | 作物生长+15%，灵茶品质+1 |
| 浓云海 | 20% | 作物生长+40%，稀有出现率+50%，灵界掉落翻倍 |
| 大潮 | 5% | 传说资源窗口 + 主线/大型事件推进 |

### 3.2 预报机制
- 每日22:00固定预报次日状态，准确率100%。
- 大潮提前7天预告。
- 预报界面同步显示“推荐作物/灵兽/活动”。

### 3.3 玩家可控机制
| 行为 | 对次日云海影响 |
|---|---:|
| 种植/收获灵茶 | +8~15 |
| 修缮/装饰建筑 | +10~25 |
| 喂养/陪伴灵兽 | +5~12 |
| 节日参与/送礼 | +6~10 |
| 破坏环境/长期不维护 | -15~-30 |

- 后期道具“云海香炉”可保底次日薄雾以上。
- 灵气越高，浓云海/大潮权重越高。

## 4. 农业、灵茶与经济系统（最终平衡）

### 4.1 经济曲线目标
| 阶段 | 日平均纯收入 | 核心来源 |
|---|---|---|
| 第1年春 | 1200~2800 | 普通作物 + 初雾绿茶 |
| 第1年夏秋 | 4500~9500 | 灵茶 + 简加工 |
| 第2年 | 15000~32000 | 加工 + 游客 + 客栈 |
| 第3年+ | 40000~80000 | 高级加工 + 大潮 + 完美山庄 |

### 4.2 普通作物定位
- 前期主力，成长更平滑，生长时间普遍缩短。
- 多次采摘作物强化（如蓝莓丛）提升稳定感。

### 4.3 灵茶定位
- 灵茶仍是核心特色，但前期收益不过强，避免碾压普通作物。
- 分层解锁：入门 -> 中级 -> 高级 -> 传说。
- 高灵气 + 浓云海时圣品概率25%~35%。

### 4.4 工坊加工链（利润放大器）
**种茶 -> 加工 -> 售卖/送礼**为中期核心循环。  
关键加工品收益倍率约2.8~3.5倍。

### 4.5 无压力经济保障
- 不会欠债，最低金钱为0。
- 每日保底“山庄补助”100。
- 客栈2级后提供稳定被动收入（3000~8000/日）。

## 5. 山庄经营与建筑升级

### 5.1 核心升级思路
- 主屋、茶园、工坊、灵兽园、温泉、观云台、客栈形成闭环。
- 每次升级不仅提升本建筑，也应影响全庄系统。

### 5.2 联动原则（关键）
- 温泉升级 -> 角色关系加成
- 茶园升级 -> 灵兽协作效率提升
- 观云台升级 -> 云海预报精度与深度提升

### 5.3 长期目标
- “完美山庄”检查清单（Perfection向）作为终局可持续目标。

## 6. 云海守护者契约系统（新增核心）

对标SDV社区中心，作为中后期长期目标系统。

### 6.1 结构
- 共6卷契约书，每卷4-6个“云海之约”。
- 完成单项得阶段奖励，完成整卷得永久全图Buff + 剧情推进。
- 无截止日期，可跨年补完。

### 6.2 六卷主题
1. 茶园苏醒契（农业灵茶）  
2. 山庄修缮契（建筑装修）  
3. 灵兽共生契（图鉴与好感）  
4. 云海羁绊契（角色社交）  
5. 灵界探索契（探索采集）  
6. 太初平衡契（终局主线）  

### 6.3 系统价值
- 明确“下一步做什么”的长期目标。
- 把分散玩法聚合成强正反馈闭环。
- 每卷完成有可见世界变化与集体反馈演出。

## 7. 角色与社交系统

### 7.1 关系框架
- 12位可婚角色，10心+求婚结构。
- 多层事件：日常对话、心事件、节日互动、婚后事件。

### 7.2 优化方向
- 婚后补充日常对话与夫妻任务，避免后期空档。
- 村民互相互动增强“活村庄”感。
- 高好感灵兽可辅助送礼触发额外社交收益。

---

## 8. 灵兽系统（轻养成修订）

### 8.1 核心改动
- 从复杂进化改为两阶段（幼年->成年），进化非强制。
- 每日只需喂食+互动各一次，负担低。
- 重点是陪伴与帮忙，而非战斗强度。

### 8.2 功能分型
- 采集型、运输型、守护型、辅助型、特殊型、传说型。
- 前期可养6只，后期12只。

### 8.3 情感反馈
- 灵兽有个性标签与日常小行为。
- 婚后配偶可协助照料，提升家庭氛围。

---

## 9. 节日系统（精简修订）

### 9.1 节日数量
- 从16个精简为12个，降低疲劳感。

### 9.2 规则
- 常规节日前3天预告，大潮前7天预告。
- 错过无惩罚，次年可补。
- 轻松参与与全力参与两档。
- 节日当天时间流速放慢。

### 9.3 季节安排
- 春3、夏3、秋3、冬2、跨季特殊1（云海大潮）。

---

## 10. 灵界与战斗系统（可选化）

### 10.1 结构
- 4个季节主题灵界区域。
- 每区分浅层/中层/深层逐步解锁。

### 10.2 核心原则
- 战斗完全可选，默认轻探索采集体验。

### 10.3 模式
- 轻松模式（默认）  
- 关闭战斗（纯采集）  
- 挑战模式（高风险高收益）  

### 10.4 惩罚
- 失败仅损失少量体力并自动回家，无严重惩罚。

---

## 11. 主线剧情与多结局

### 11.1 主线框架
- 全6章，从“归山重建”推进到“界桥终局抉择”。
- 主线与云海、灵气、角色好感、契约进度深度耦合。

### 11.2 结局设计
- 真结局 + 单身守护 + 多角色线结局 + Bad End + 隐藏结局。
- 平衡路线为最高难度，要求高灵气与高社交完成度。

## 12. 数值与难度框架

### 12.1 时间与体力
- 体力初始120，上限350（修订版）。
- 疲劳只减速不强制结束，保持无压力。

### 12.2 灵气阶段
- 荒废 -> 苏醒 -> 兴盛 -> 繁华 -> 太初。
- 影响作物品质、云海状态、事件池与终局条件。

### 12.3 难度模式
- 轻松：体力更宽松、成长更快
- 标准：默认体验
- 挑战：衰减加快、资源更紧
- 自由模式：可关闭主线，仅经营与社交

## 16. 技术与生产提效优化（小团队重点）

### 16.1 数据驱动与逻辑解耦
- 核心原则：基于 `Excel/CSV` 控制数值与配置，不在引擎内硬编码业务参数。
- 主配置表示例字段：`item_id`、`base_price`、`growth_days`、`cloud_bonus`、`aura_requirement`。
- 建议覆盖范围：作物、灵茶、节日、契约、NPC对话触发条件、灵兽行为参数。
- 预期收益：平衡迭代改表即可生效，避免“牵一发而动全身”。

### 16.2 插件化架构（Godot 4建议）
- 对话系统：使用 `Dialogue Manager` 或 `Ink`，剧情内容与代码逻辑分离。
- 状态机系统：建立统一FSM（NPC与灵兽共用框架）。
- 通用状态建议：巡逻、互动、跟随、工作、休息、特殊事件。
- 技术收益：减少重复脚本，便于后期扩内容与修Bug。

### 16.3 美术资产提效：云海视觉规范
- 建立季节色彩分层（Palette Swapping）。
- 同类资产优先“一套基底图 + Shader调色”，减少四季重复绘制。
- 目标收益：同类资产绘制成本降低，风格一致性更强。

### 16.4 云雾三层视觉结构
- 远景层：远山云海，视差滚动强化空间感。
- 环境层：地表薄雾粒子，受风向脚本控制。
- 交互层：角色经过时散开的动态云块。
- 目标效果：保证“治愈氛围”同时提升交互反馈。

### 16.5 数值心流轨迹设计
- 前期（第1个月）：高频小奖励（新种子、新互动、新提示）。
- 中期（第1年末）：阶段大目标（客栈开业、灵兽成年、建筑联动）。
- 后期（契约系统）：世界改变型奖励（景观、全图Buff、终局路线）。
- 实施建议：维护一份“压力-回报呼吸表”，提前发现无聊点。

### 16.6 大潮稀缺性与期待感
- 设计“大潮专属NPC/商店/订单池”，仅在大潮窗口出现。
- 提前7天预告，让玩家有明确准备周期。
- 设计目标：在日常经营中持续提供“近在眼前的期待事件”。
