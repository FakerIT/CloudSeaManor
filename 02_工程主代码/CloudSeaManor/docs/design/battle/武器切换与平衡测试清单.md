# 武器切换与平衡测试清单

## 一、功能链路验证（装备 -> 合并 -> 生效）

- [ ] 启动后读取 `weapon_table.csv` 成功，至少存在 1 把武器
- [ ] 调用 `BattleManager::SetEquippedWeapon()` 可成功切换到目标武器
- [ ] 进入战斗后 `BattlePlayerState.equipped_weapon_id` 与设置一致
- [ ] 切换武器后 `purification_rate / crit_chance / energy_recover_rate / skill_cooldown_scale` 立即变化
- [ ] 技能冷却总时长随 `skill_cooldown_scale` 变化，当前剩余冷却按比例平滑过渡

## 二、实时战斗效果验证

- [ ] 技能伤害公式包含 `weapon_attack_power`（同技能下高攻武器伤害明显更高）
- [ ] 暴击触发概率受 `crit_chance_bonus` 影响（100 次采样接近预期区间）
- [ ] 暴击伤害受 `crit_multiplier_bonus` 影响（暴击伤害倍率变化正确）
- [ ] 回能速率受 `energy_recover_bonus` 影响（每秒回能变化可观测）
- [ ] 以上效果在战斗中切换武器后无需重开战斗即可生效

## 三、平衡性基线（建议）

- [ ] T1 武器（quality=1）通关外围战斗时间在 60~90 秒
- [ ] T3 武器（quality=3）相对 T1 平均耗时下降 15%~25%
- [ ] T5 武器（quality=5）不应使 BOSS 战无脑碾压（建议耗时下降不超过 35%）
- [ ] 能量系统保持“有技能窗口 + 空窗期”节奏，避免无限技能循环
- [ ] 低品质武器仍有可玩性（不出现“伤害过低导致战斗拖沓”）

## 四、异常与回退

- [ ] 传入不存在的武器 ID：返回失败，不崩溃，输出 Warning
- [ ] 武器表为空时回退默认战斗参数（可正常进入战斗）
- [ ] 数值越界（负暴击率、极端冷却缩放）被限制在安全区间
- [ ] 战斗中切换武器时不会出现 NaN/负冷却/能量超界
