## 一、设计理念

### 1.1 核心理念："净化"而非"击杀"

云海山庄的战斗系统以**净化（Purify）**为核心机制，区别于传统游戏的"击杀"模式：

- 污染灵体（Polluted Spirit）是失去灵性的自然生灵，不是敌人
- 玩家的目标是将污染值（Purity）从 0% 恢复至 100%，而非消灭
- 净化完成时灵体恢复清明并给予奖励，失败时灵体暂时退散（无任何惩罚）
- 战斗以**控场、减伤、回复、增益**为主旋律，策略深度足够但休闲不肝

### 1.2 设计目标

| 目标 | 说明 |
|------|------|
| 深度策略 | 天气联动 + 羁绊加成 + 技能克制，提供足够的策略空间 |
| 轻松体验 | 无死亡惩罚、无逃跑惩罚、无负面效果，可随时退出 |
| 视觉享受 | 全程云海仙侠风格渲染，净化特效华丽 |
| 性能优先 | 战斗场景独立渲染层，60 FPS 流畅运行 |
| 数据驱动 | 所有敌人、技能、事件由外部数据表驱动，无需硬编码 |

## 四、战斗流程

### 4.1 进入战斗

```
触发条件：
  1. 玩家接近污染灵体（碰撞检测距离 < 60px）
  2. 玩家主动与污染灵体交互（E键）
  3. 剧情事件触发
  4. 灵界区域强制遭遇

进入流程：
  1. 检测灵体类型（普通/精英/BOSS）
  2. 冻结主世界移动（暂停player.Update）
  3. 切换战斗渲染层（叠加在主世界上方）
  4. 加载战场背景（云雾/灵界/矿洞等）
  5. 初始化灵体位置（从主世界坐标映射）
  6. 加载灵兽出战数据（羁绊等级、技能列表）
  7. 计算初始天气倍率（CloudSystem）
  8. 显示战斗UI面板
  9. 战斗状态切换为 Active
```

### 4.2 战斗回合循环（60 FPS 实时制，非传统回合制）

```
每帧 BattleManager::Update()：

  1. [输入处理] 处理玩家技能释放输入
     ↓
  2. [灵兽AI] 更新出战灵兽技能AI
     - 检查冷却
     - 选择目标
     - 释放技能
     ↓
  3. [敌人AI] 更新污染灵体行为
     - 状态机更新（待机/接近/攻击）
     - 位置移动（朝向玩家）
     - 技能释放（根据类型和冷却）
     ↓
  4. [效果处理] 处理所有进行中的效果
     - Buff/Debuff 计时
     - 持续伤害（污染加深）
     - 持续治疗
     - 护盾衰减
     ↓
  5. [碰撞检测] 检测技能命中
     - 圆形碰撞（玩家技能范围 vs 灵体位置）
     - 触发命中回调
     ↓
  6. [污染值更新] 更新灵体污染值
     - 计算本次净化量
     - 检查是否净化完成
     - 更新状态
     ↓
  7. [胜负判定]
     - 全部灵体净化 → Victory
     - 玩家撤退 → Retreat
     - 玩家超时（无惩罚机制，此条件实际不存在）
     ↓
  8. [UI更新] 渲染所有战斗UI元素
     - 灵体血条
     - 玩家能量条
     - 技能按钮状态
     - 战斗日志
```

### 4.3 技能释放流程

```
玩家按下技能键（Q/W/E/R 对应4个快捷技能）：

  1. 检查能量是否充足
  2. 检查冷却是否完成
  3. 播放技能释放动画（粒子特效）
  4. 消耗能量
  5. 开始冷却计时
  6. 记录 BattleAction
  7. 计算技能效果
     - 基础效果值 × 净化效率 × 天气倍率 × 羁绊加成
     - 判定是否暴击
  8. 应用效果
     - 对单体：直接削减目标污染值
     - 对范围：遍历范围内所有灵体
     - 对自己：添加buff/恢复能量
  9. 生成战斗日志
  10. 触发命中特效（粒子系统）
```

### 4.4 净化完成判定

```
污染值归零判定：

  if (spirit.current_pollution <= 0.0f) {
      spirit.is_defeated = true;
      // 播放净化成功特效（粒子爆炸→消散）
      // 显示净化奖励（经验+物品）
      // 记录到 BattleLog
      // 检查是否全部净化完成
  }

净化量计算：
  污染值削减 = 技能基础净化量
              × 净化效率修正（player.purification_rate）
              × 天气倍率（CloudState对应值）
              × 羁绊加成（1 + heart_level × 0.1）
              × 元素克制（克制元素+0.5，否则1.0）
              × 暴击修正（暴击时×1.5）

精英/BOSS难度修正：
  精英：污染值 = 200%，净化难度 ×1.5
  BOSS：污染值 = 300%，净化难度 ×2.0
```

### 4.5 战斗结束

```
Victory（全部净化完成）:
  1. 播放胜利动画（云海汇聚特效）
  2. 显示结算面板：
     - 净化灵体数量
     - 获得净化经验
     - 获得物品奖励
     - 羁绊好感提升（灵兽）
  3. 延迟3秒后淡出
  4. 战斗状态 → Inactive
  5. 恢复主世界移动
  6. 发放奖励到 Inventory

Retreat（玩家主动撤退）:
  1. 显示"暂避锋芒"提示
  2. 战斗状态 → Inactive
  3. 恢复主世界移动
  4. 灵体进入"暂时退散"状态（3分钟后重新出现）
  5. 无任何惩罚

（注意：没有"战斗失败"状态——这是设计核心）
```

```cpp
void CommonSpiritAI(PollutedSpirit& spirit, float delta) {
    // 待机 → 接近 → 攻击 → 冷却 → 接近 → ...
    float dist = Distance(spirit.position, player.position);
    if (dist > spirit.attack_range) {
        spirit.state = SpiritAIState::Approach;
        MoveToward(spirit, player.position, spirit.speed * delta);
    } else if (spirit.attack_timer <= 0.0f) {
        spirit.state = SpiritAIState::Attack;
        ExecuteAttack(spirit);
        spirit.attack_timer = spirit.attack_cooldown;
    }
}
```

## 八、灵兽战斗协同

### 8.1 出战机制

- 最多3只灵兽同时出战
- 每只灵兽在战场上独立行动（AI控制）
- 灵兽自动选择最优技能释放
- 玩家可通过快捷键手动指挥灵兽释放特定技能

### 8.2 灵兽AI决策

```cpp
// 灵兽技能AI伪代码
void SpiritBeastAI(BattlePartner& partner, float delta) {
    // 遍历所有技能
    for (int i = 0; i < partner.active_skill_ids.size(); i++) {
        if (cooldowns[i] > 0.0f) continue;  // 冷却中
        if (energy < skill.energy_cost) continue;  // 能量不足

        // 决策逻辑
        if (skill.type == SkillType::Active) {
            // 优先攻击污染值最高的敌人
            auto target = SelectHighestPollutionTarget();
            if (target != nullptr) {
                CastSkill(skill, target);
                break;
            }
        }
    }
}
```

## 九、渲染设计

### 9.1 渲染层次

| 层次 | 内容 | 说明 |
|------|------|------|
| L0 | 主世界 | 战斗前已冻结的画面 |
| L1 | 战场背景 | 云雾渐变/灵界场景/矿洞背景 |

### 9.2 污染灵体视觉效果

- 普通灵体：透明发光球体，颜色根据元素变化
- 精英灵体：体积更大，光晕更强，带旋转外环
- BOSS灵体：巨大，带多层次光晕，带污染粒子持续飘散
- 净化中：颜色从深紫/深灰逐渐变亮
- 净化完成：粒子爆炸→灵体变白→消散为光点

### 10.2 存档时战斗状态处理

```
存档检测到 in_battle == true 时：
  1. 强制执行 Victory（自动完成所有剩余灵体的净化）
  2. 发放全部奖励
  3. in_battle = false
```
