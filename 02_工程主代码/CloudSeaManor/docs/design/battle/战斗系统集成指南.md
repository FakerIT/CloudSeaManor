# 战斗系统集成指南

> 文档版本：v1.0 | 状态：设计阶段 | 用途：指导程序员将战斗系统集成到游戏主循环

---

## 一、集成步骤概览

```
Step 1: 添加头文件引用
Step 2: 在 GameWorldState 中添加战斗状态
Step 3: 在 GameApp 主循环中集成 BattleManager
Step 4: 配置输入处理（技能释放快捷键）
Step 5: 挂载战斗触发（接近敌人自动进入）
Step 6: 连接存档系统
Step 7: 配置战斗区域和敌人刷新
```

### Step 3: 在 GameApp 主循环中集成

在 `GameApp.cpp` 的 `Update` 方法中，当检测到战斗触发时切换模式：

```cpp
// GameApp.hpp 新增成员
private:
    CloudSeamanor::engine::BattleManager battle_manager_;
    bool in_battle_mode_ = false;

// GameApp.cpp Update 方法中添加：
void GameApp::Update(float delta_seconds) {
    if (in_battle_mode_) {
        // 战斗模式：冻结主世界更新，仅更新战斗系统
        battle_manager_.Update(delta_seconds,
            world_state_.GetPlayer().GetPosition().x,
            world_state_.GetPlayer().GetPosition().y);

        if (!battle_manager_.IsInBattle()) {
            // 退出战斗，恢复主世界
            in_battle_mode_ = false;
        }
        return; // 跳过主世界更新
    }

    // ---- 正常的主世界 Update ----
    // （现有的所有 Update 逻辑保持不变）
    if (auto* spirit_sys = dynamic_cast<SpiritBeastSystem*>(FindSystem("SpiritBeast"))) {
        spirit_sys->Update(world_state_, delta_seconds);
    }

    // ... 其余逻辑 ...

    // 在主世界循环中检测战斗触发
    CheckBattleTrigger_();
}
```

```cpp
        battle_manager_.OnSkillKeyPressed(0, mouse_position.x, mouse_position.y);
    }
    if (sf::Keyboard::isKeyPressed(sf::Keyboard::W)) {
        battle_manager_.OnSkillKeyPressed(1, mouse_position.x, mouse_position.y);
    }
    if (sf::Keyboard::isKeyPressed(sf::Keyboard::E)) {
        battle_manager_.OnSkillKeyPressed(2, mouse_position.x, mouse_position.y);
    }
    if (sf::Keyboard::isKeyPressed(sf::Keyboard::R)) {
        battle_manager_.OnSkillKeyPressed(3, mouse_position.x, mouse_position.y);
    }

    // 暂停：Escape
    if (sf::Keyboard::isKeyPressed(sf::Keyboard::Escape)) {
        battle_manager_.Pause();
        battle_manager_.GetUI().SetPauseMenuVisible(true);
    }

    // 撤退：X
    if (sf::Keyboard::isKeyPressed(sf::Keyboard::X)) {
        battle_manager_.Retreat();
    }

    return; // 跳过主世界输入处理
}
```

## 三、渲染集成

在 `GameApp::Render` 中添加战斗层：

```cpp
void GameApp::Render(sf::RenderWindow& window) {
    // ---- 主世界渲染（保持不变）----
    DrawWorld(window);
    DrawEntities(window);
    DrawUI(window);

    // ---- 战斗层渲染（战斗模式时）----
    if (in_battle_mode_) {
        // 渲染战场背景叠加
        DrawBattleBackground(window);

        // 渲染污染灵体
        DrawPollutedSpirits(window);

        // 渲染战斗UI
        battle_manager_.GetUI().Draw(window);
    }

    // ---- 结算面板（胜利时）----
    if (battle_manager_.CurrentState() == BattleState::Victory) {
        DrawVictoryPanel(window);
    }
}
```

## 五、灵兽系统集成

### 5.1 从 SpiritBeastSystem 获取出战数据

在 `BattleManager::LoadPartnersFromSpiritBeasts` 中：

```cpp
void BattleManager::LoadPartnersFromSpiritBeasts(
    const std::vector<std::string>& spirit_beast_ids
) {
    // 获取 SpiritBeastSystem 实例
    auto* spirit_sys = GetSystem<SpiritBeastSystem>("SpiritBeast");

    for (const auto& id : spirit_beast_ids) {
        const auto& beast_data = spirit_sys->GetSpiritBeastData(id);

        BattlePartner partner;
        partner.spirit_beast_id = id;
        partner.name = beast_data.display_name;
        partner.heart_level = beast_data.heart_level;
        partner.purification_rate_mod = 1.0f + beast_data.heart_level * 0.1f;

        // 从灵兽技能数据加载战斗技能
        for (const auto& skill_id : beast_data.battle_skill_ids) {
            partner.active_skill_ids.push_back(skill_id);
            partner.cooldown_total.push_back(GetSkillCooldown(skill_id));
        }

        field_.AddPartner(partner);
    }
}
```

### 5.2 战斗后更新羁绊好感

在 `ProcessVictory_` 中：

```cpp
void BattleManager::ProcessVictory_() {
    const auto& result = field_.GetResult();

    auto* spirit_sys = GetSystem<SpiritBeastSystem>("SpiritBeast");

    // 更新灵兽羁绊
    for (const auto& [beast_id, favor_delta] : result.partner_favor_gained) {
        spirit_sys->AddFavor(beast_id, favor_delta);
    }

    // 发放净化经验到 SpiritGuard 技能
    auto* skill_sys = GetSystem<SkillSystem>("Skill");
    skill_sys->AddExp(SkillType::SpiritGuard, result.total_exp_gained);

    field_.EndBattle();
    state_ = BattleState::Inactive;
}
```

## 六、性能注意事项

1. **战斗不活跃时禁用渲染**：当 `BattleState::Inactive` 时，不渲染任何战斗相关元素
2. **粒子特效池化**：使用对象池复用战斗粒子，避免频繁 `new/delete`
3. **敌人数量限制**：单场战斗最多同时存在10只敌人
4. **UI脏标记**：仅在数据变化时更新UI，避免每帧重建
