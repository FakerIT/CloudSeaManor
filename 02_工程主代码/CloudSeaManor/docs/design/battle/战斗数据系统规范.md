# 战斗数据系统规范（怪物 / 武器 / 数值）

## 目标

- 将战斗核心数据从硬编码迁移到表驱动。
- 明确三类数据边界：怪物信息、武器信息、全局数值。
- 保证策划可直接改表，程序自动加载并校验。

## 数据文件

- `assets/data/battle/spirit_table.csv`：怪物信息（污染灵体）
- `assets/data/battle/weapon_table.csv`：武器结构与战斗修正
- `assets/data/battle/balance_table.csv`：全局战斗数值
- `assets/data/battle/skill_table.csv`：技能信息
- `assets/data/battle/zone_table.csv`：战斗区域信息

## 怪物信息结构（MonsterTableEntry）

- `id`：唯一 ID
- `name`：显示名
- `zone`：所属区域
- `star`：星级
- `type`：`common|elite|boss`
- `element`：`neutral|water|fire|wood|metal|earth|light|dark`
- `pollution`：最大污染值
- `health`：基础生命
- `speed`：移动速度
- `attack_cd`：攻击间隔
- `attack_dmg`：攻击伤害
- `reward_exp`：经验奖励
- `rewards`：掉落（分号分隔，支持 `item*count` 语法）
- `behavior_type`：行为标签（用于 AI 扩展）

## 武器结构（WeaponTableEntry）

- `id`：唯一 ID
- `name`：显示名
- `weapon_type`：`melee|ranged|caster...`
- `element`：武器元素
- `base_attack`：基础攻击
- `purify_rate_bonus`：净化倍率加成
- `crit_chance_bonus`：暴击率加成
- `crit_multiplier_bonus`：暴击伤害加成
- `energy_recover_bonus`：能量回复加成
- `skill_cooldown_scale`：技能冷却缩放（<1 更快）
- `quality`：品质

## 全局数值结构（BattleTuningConfig）

- `player_base_crit_chance`
- `player_base_purify_rate`
- `partner_base_skill_power`
- `partner_default_cooldown`
- `area_skill_hit_radius`

## 程序接入点

- `BattleManager::Initialize()` 加载全部表
- `BattleField` 维护：
  - `monster_table_`
  - `weapon_table_`
  - `tuning_config_`
- 战斗计算优先使用 `tuning_config_`，避免再次引入魔法数。

## 校验约束（建议）

- `id` 不允许重复
- 数值字段非法时跳过该行并输出日志
- `type/element` 非法值回退默认值
- 关键表加载失败时保留旧值并输出 Warning
