## Phase 1（最高优先级）告白 / 婚礼系统 MVP

### P1-SOC-001 | 关系状态机（domain）+ 可持久化字段

**进度标识**：✅ 已完成（2026-04-28）

**任务目的**：
- 建立单伴侣（MVP）关系终局状态机：告白 → 订婚 → 预约婚礼 → 婚礼日 → 婚后
- 状态字段可存档/读档，旧存档可用默认值兼容读取

**技术要求**：
- 领域层实现（不依赖 engine / IO）
- 状态迁移 API 清晰、可测试（纯函数/小对象，避免散落在交互逻辑中）

**交付标准**：
- 任意存档槽位保存/读取后，关系字段一致
- 婚礼日到来时，状态能正确从 `WeddingScheduled` 迁移到 `Married`

### P1-SOC-002 | 对话注入：聊天菜单追加“告白/预约婚礼”选项

**进度标识**：✅ 已完成（2026-04-28）

**任务目的**：
- 在与 NPC 对话开始前，动态注入一个“关系菜单节点”，让玩家可选择“继续聊天 / 告白 / 预约婚礼”
- 选择“告白/预约婚礼”后触发状态机推进与资源扣除

**技术要求**：
- 不修改 `DialogueEngine` 的路由规则；使用 `on_node_change` 进入 action 节点时执行副作用
- 心事件对话不注入（避免重入/打断剧情）

**交付标准**：
- 满足条件时能看到“告白”选项；不满足时不会出现
- 订婚后能看到“预约婚礼”选项并成功安排日期

### P1-SOC-003 | 婚礼日事件（engine）+ 当日结算提示

**进度标识**：✅ 已完成（2026-04-28）

**任务目的**：
- 当天数到达预约婚礼日时，触发婚礼日事件：关系状态变更、提示文本、必要的系统联动（如 NPC married 标记）

**技术要求**：
- 接入 `GameRuntime::OnDayChanged()` 的每日结算链路
- 与节日系统冲突时：预约日期自动顺延（MVP 允许固定规则）

**交付标准**：
- 存档读档后婚礼日仍能触发（不会丢失）
- 触发后状态变为 `Married`，并有明确 UI 提示

### P1-SOC-004 | 婚后增益（domain+engine）：每日增益与 BuffSystem 对接

**进度标识**：✅ 已完成（2026-04-28）

**任务目的**：
- 婚后提供“每日稳定增益”（MVP）：体力消耗略降 / 恢复略升（通过 RuntimeBuff 每日刷新实现）

**技术要求**：
- 领域层定义增益策略，engine 负责在每日结算时应用

**交付标准**：
- 结婚后每日开始都会重新获得 buff（持续 24h）

### P1-SOC-005 | 存档版本升级与兼容迁移（v8）

**进度标识**：✅ 已完成（2026-04-28）

**任务目的**：
- 为关系系统新增存档行 `relationship|...`，并将存档版本升级到 v8
- v7 及更早存档读入时保持默认关系状态，不崩溃

**技术要求**：
- 不破坏已有 plot schema / checksum / 原子写入流程

**交付标准**：
- v7 存档能成功读入并继续存成 v8

### P11-GAME-001 | 灵气可视化与潮汐预报 MVP

**进度标识**：⏳ 待开始

**执行清单（可勾选）**：
- [ ] 在庄园入口或茶室增加 `灵泉井` 可视化反馈
- [ ] 井水颜色或粒子表现映射当前灵气阶段
- [ ] 起床时增加短暂灵气状态边缘特效
- [ ] 在日历或挂历中标注 `大潮日 / 小潮日`
- [ ] 大潮日与小潮日接入经营/战斗收益修正

**数据表草案**：
- [ ] `assets/data/ecology/spirit_tide_calendar.csv`
- [ ] 列：`DayOfMonth, TideType, AuraModifier, CombatModifier, GrowthModifier, Note`

**分层落点清单**：
- `domain`：灵气阶段、潮汐类型、收益修正
- `engine`：灵泉井表现、起床光晕、日历 UI 标注
- `infrastructure`：潮汐日历表加载

**交付标准**：
- [ ] 玩家无需调试面板即可感知庄园灵气状态
- [ ] 玩家可提前知道大潮日并据此安排日程
