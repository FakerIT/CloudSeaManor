## 版本标签使用指南

### 创建发布

```bash
# 创建标签
git tag -a v0.2.0 -m "v0.2.0: MVP 功能完成"

# 推送标签
git push origin v0.2.0
```

### 在 Issue 中使用

在 Issue 或 PR 中添加版本标签：
- `version:0.2.0` - 计划在 v0.2.0 中完成
- `version:0.3.0` - 计划在 v0.3.0 中完成

### 查看版本进度

使用 GitHub Milestones 查看每个版本的 Issue 进度。

## 发布检查清单

每个版本发布前必须确认：

- [ ] 所有计划内的 Issue 已关闭
- [ ] 代码通过所有 CI 检查
- [ ] 单元测试全部通过
- [ ] 手动测试验证关键功能
- [ ] CHANGELOG.md 已更新
- [ ] 版本号已更新
- [ ] 存档格式已确定（如有变更需迁移）
