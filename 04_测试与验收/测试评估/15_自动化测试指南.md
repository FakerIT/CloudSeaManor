## 一、自动化测试策略

### 1.1 测试金字塔

```
        ╱  E2E测试  ╲          ← 少量,模拟完整用户操作
       ╱  (手动为主)  ╲
      ╱──────────────╲
     ╱   集成测试      ╲       ← 适量,系统间联动
    ╱   (半自动化)      ╲
   ╱────────────────────╲
  ╱     单元测试          ╲     ← 大量,领域逻辑验证
 ╱     (完全自动化)        ╲
╱──────────────────────────╲
```

### 1.2 自动化优先级

| 层级 | 自动化程度 | 覆盖范围 | 工具 |
|------|----------|---------|------|
| 领域层(domain) | 完全自动化 | 纯逻辑验证 | Catch2 |
| 引擎层(engine) | 半自动化 | 运行时协调 | Catch2 + 手动 |
| 基础设施层(infrastructure) | 完全自动化 | 数据解析验证 | Catch2 |
| UI/交互 | 手动为主 | 视觉/体验验证 | 人工 |

## 二、单元测试框架搭建

### 2.1 推荐框架: Catch2

**选择理由**:
- C++20兼容
- 头文件即可使用,集成简单
- 表达式分解,错误信息清晰
- BDD风格支持
- SFML项目广泛使用

### 2.2 集成步骤

#### Step 1: 添加Catch2到CMakeLists.txt

```cmake
# 在CMakeLists.txt中添加测试目标
include(FetchContent)
FetchContent_Declare(
    Catch2
    GIT_REPOSITORY https://github.com/catchorg/Catch2.git
    GIT_TAG v3.5.2
)
FetchContent_MakeAvailable(Catch2)

# 测试可执行文件
add_executable(CloudSeamanor_tests
    tests/TestRunner.cpp
    tests/domain/PlayerTest.cpp
    tests/domain/InventoryTest.cpp
    tests/domain/GameClockTest.cpp
    tests/domain/StaminaTest.cpp
    tests/domain/CloudSystemTest.cpp
    tests/domain/FestivalSystemTest.cpp
    tests/domain/ContractTest.cpp
    tests/domain/SkillSystemTest.cpp
    tests/domain/WorkshopSystemTest.cpp
    tests/infrastructure/GameConfigTest.cpp
)

target_link_libraries(CloudSeamanor_tests
    PRIVATE
        CloudSeamanor_core
        Catch2::Catch2WithMain
)

include(CTest)
include(Catch)
catch_discover_tests(CloudSeamanor_tests)
```

#### Step 2: 创建测试运行器

```cpp
// tests/TestRunner.cpp
#define CATCH_CONFIG_MAIN
#include <catch2/catch_test_macros.hpp>
```

#### Step 3: 编写测试用例示例

```cpp
// tests/domain/InventoryTest.cpp
#include <catch2/catch_test_macros.hpp>
#include "CloudSeamanor/Inventory.hpp"

using namespace CloudSeamanor::domain;

TEST_CASE("Inventory - 添加物品", "[inventory]") {
    Inventory inv;

    SECTION("添加单个物品") {
        inv.AddItem("TeaLeaf", 5);
        REQUIRE(inv.CountOf("TeaLeaf") == 5);
    }

    SECTION("叠加同类物品") {
        inv.AddItem("TeaLeaf", 3);
        inv.AddItem("TeaLeaf", 2);
        REQUIRE(inv.CountOf("TeaLeaf") == 5);
    }

    SECTION("添加空ID忽略") {
        inv.AddItem("", 5);
        REQUIRE(inv.TotalItemCount() == 0);
    }

    SECTION("添加0数量忽略") {
        inv.AddItem("TeaLeaf", 0);
        REQUIRE(inv.CountOf("TeaLeaf") == 0);
    }
}

TEST_CASE("Inventory - 移除物品", "[inventory]") {
    Inventory inv;
    inv.AddItem("TeaLeaf", 10);

    SECTION("正常移除") {
        REQUIRE(inv.RemoveItem("TeaLeaf", 3));
        REQUIRE(inv.CountOf("TeaLeaf") == 7);
    }

    SECTION("移除全部") {
        REQUIRE(inv.RemoveItem("TeaLeaf", 10));
        REQUIRE(inv.CountOf("TeaLeaf") == 0);
    }

    SECTION("移除超过持有量失败") {
        REQUIRE_FALSE(inv.RemoveItem("TeaLeaf", 15));
        REQUIRE(inv.CountOf("TeaLeaf") == 10);
    }
}
```

## 三、领域层自动化测试用例

### 3.1 GameClock 测试用例

```cpp
TEST_CASE("GameClock - 时间推进", "[clock]") {
    GameClock clock;
    clock.Initialize(1, 360.0f); // 第1天, 6:00

    SECTION("Tick推进时间") {
        clock.Tick(60.0f); // 推进60分钟
        REQUIRE(clock.Hour() == 7);
    }

    SECTION("跨日处理") {
        clock.Tick(1080.0f); // 推进18小时到24:00
        REQUIRE(clock.Day() == 2);
    }

    SECTION("季节计算") {
        clock.Initialize(28, 360.0f); // 第28天=春季最后一天
        REQUIRE(clock.Season() == Season::Spring);
        clock.Initialize(29, 360.0f); // 第29天=夏季第1天
        REQUIRE(clock.Season() == Season::Summer);
    }

    SECTION("时段划分") {
        clock.Initialize(1, 360.0f);  // 6:00
        REQUIRE(clock.CurrentPhase() == DayPhase::Morning);
        clock.Initialize(1, 840.0f);  // 14:00
        REQUIRE(clock.CurrentPhase() == DayPhase::Afternoon);
        clock.Initialize(1, 1080.0f); // 18:00
        REQUIRE(clock.CurrentPhase() == DayPhase::Evening);
        clock.Initialize(1, 1320.0f); // 22:00
        REQUIRE(clock.CurrentPhase() == DayPhase::Night);
    }
}
```

### 3.2 Stamina 测试用例

```cpp
TEST_CASE("Stamina - 体力系统", "[stamina]") {
    StaminaSystem stamina;

    SECTION("新手保护期消耗减半") {
        stamina.Initialize(100.0f, 200.0f, 14, 0.7f);
        stamina.SetGameDay(1);
        stamina.Consume(10.0f);
        REQUIRE(stamina.Current() == Catch::Approx(93.0f)); // 100 - 10*0.7
    }

    SECTION("保护期后正常消耗") {
        stamina.Initialize(100.0f, 200.0f, 14, 0.7f);
        stamina.SetGameDay(15);
        stamina.Consume(10.0f);
        REQUIRE(stamina.Current() == Catch::Approx(90.0f)); // 100 - 10
    }

    SECTION("体力不低于0") {
        stamina.Initialize(5.0f, 200.0f, 14, 0.7f);
        stamina.ConsumeRaw(100.0f);
        REQUIRE(stamina.Current() == 0.0f);
    }

    SECTION("体力不超过上限") {
        stamina.Initialize(195.0f, 200.0f, 14, 0.7f);
        stamina.Recover(100.0f);
        REQUIRE(stamina.Current() == 200.0f);
    }
}
```

### 3.3 CloudSystem 测试用例

```cpp
TEST_CASE("CloudSystem - 云海系统", "[cloud]") {
    CloudSystem cloud;

    SECTION("新手保护天气") {
        // 前14天不会出现晴空
        for (int day = 1; day <= 14; ++day) {
            cloud.AdvanceToNextDay(day);
            REQUIRE(cloud.CurrentState() != CloudState::Clear);
        }
    }

    SECTION("灵气增益正确") {
        cloud.SetStates(CloudState::Clear, CloudState::Clear);
        REQUIRE(cloud.SpiritEnergyGain() == 5);

        cloud.SetStates(CloudState::Mist, CloudState::Mist);
        REQUIRE(cloud.SpiritEnergyGain() == 12);

        cloud.SetStates(CloudState::DenseCloud, CloudState::DenseCloud);
        REQUIRE(cloud.SpiritEnergyGain() == 25);

        cloud.SetStates(CloudState::Tide, CloudState::Tide);
        REQUIRE(cloud.SpiritEnergyGain() == 80);
    }

    SECTION("预报22:00后可见") {
        cloud.UpdateForecastVisibility(1, 21);
        REQUIRE_FALSE(cloud.IsForecastVisible());
        cloud.UpdateForecastVisibility(1, 22);
        REQUIRE(cloud.IsForecastVisible());
    }
}
```

## 四、数值验证自动化

### 4.1 经济曲线模拟器

```cpp
TEST_CASE("经济曲线 - 第1年春季日收入", "[economy]") {
    // 模拟第1年春季的日收入
    // GDD目标: 1200~2800
    float daily_income = 0.0f;

    // 假设: 种植4块茶田,每块产出5个茶叶,每个售价60
    daily_income = 4 * 5 * 60.0f; // = 1200

    REQUIRE(daily_income >= 1200.0f);
    REQUIRE(daily_income <= 2800.0f * 1.15f); // 允许+15%偏差
}
```

### 4.2 天气概率验证

```cpp
TEST_CASE("云海概率 - 天气分布验证", "[cloud][numerical]") {
    CloudSystem cloud;
    int clear_count = 0, mist_count = 0, dense_count = 0, tide_count = 0;
    const int total_days = 1000;

    for (int day = 15; day < 15 + total_days; ++day) {
        cloud.AdvanceToNextDay(day);
        switch (cloud.CurrentState()) {
            case CloudState::Clear: ++clear_count; break;
            case CloudState::Mist: ++mist_count; break;
            case CloudState::DenseCloud: ++dense_count; break;
            case CloudState::Tide: ++tide_count; break;
        }
    }

    // 验证概率在允许偏差内
    float clear_ratio = static_cast<float>(clear_count) / total_days;
    float mist_ratio = static_cast<float>(mist_count) / total_days;
    float dense_ratio = static_cast<float>(dense_count) / total_days;
    float tide_ratio = static_cast<float>(tide_count) / total_days;

    REQUIRE(clear_ratio == Catch::Approx(0.40).margin(0.15));
    REQUIRE(mist_ratio == Catch::Approx(0.35).margin(0.15));
    REQUIRE(dense_ratio == Catch::Approx(0.20).margin(0.15));
    REQUIRE(tide_ratio == Catch::Approx(0.05).margin(0.08));
}
```

## 五、CI集成建议

### 5.1 GitHub Actions 配置

```yaml
# .github/workflows/test.yml
name: Tests

on:
  push:
    branches: [ main, develop ]
  pull_request:
    branches: [ main ]

jobs:
  test:
    runs-on: windows-latest
    steps:
      - uses: actions/checkout@v4

      - name: Configure CMake
        run: cmake --preset vs2022-x64

      - name: Build
        run: cmake --build --preset build-debug

      - name: Run Unit Tests
        working-directory: build/vs2022-x64
        run: ctest --output-on-failure
```

### 5.2 本地测试命令

```powershell
# 配置
cmake --preset vs2022-x64

# 构建
cmake --build --preset build-debug

# 运行测试
cd build/vs2022-x64
ctest --output-on-failure

# 运行特定测试
.\Debug\CloudSeamanor_tests.exe "[inventory]"
.\Debug\CloudSeamanor_tests.exe "[clock]"
```
